FSC Test Report: Tomato Project

1. Component Testing

During the development of the Tomato project, we conducted component testing to ensure that each individual part of the app was functioning as intended before integrating them into the whole system. We used JUnit to perform unit testing on the various classes and methods that were created. We created tests for adding, editing, and deleting reminders, as well as for setting and canceling notifications. We also tested the functionality of the app’s UI components, such as the buttons, text fields, and radio buttons.


To select the tests and test data, we followed a structured approach based on the expected requirements. We created test cases for each function and method of the app, taking into account possible edge cases and scenarios that could occur. We also created test data to ensure that the app could handle different types of inputs and outputs. Our choices were good as they covered all the requirements of the app, and helped us to identify and fix any bugs or issues.


2. System Testing

After conducting component testing, we performed system testing to test the integration of all the individual components into the complete app. We tested the functionality of the app as a whole, including adding, editing, and deleting reminders, and setting and canceling notifications. We also tested the app’s user interface and user experience to ensure that it was user-friendly and easy to use.


We used manual testing to perform system testing and selected test cases and test data that covered all the expected requirements of the app. We also used the Android Studio Emulator to test the app on different devices and screen sizes. We documented the test results and any issues that we encountered during testing, and used them to improve the app’s performance and functionality.


3. Acceptance Testing

During the system testing phase, we also conducted acceptance testing to ensure that the app met the acceptance criteria on our user stories. We used the acceptance criteria to drive our system testing and ensure that each user story was completely implemented. The user stories we tested included:


- As a user, I want to be able to add a reminder to the app.

- As a user, I want to be able to edit and delete reminders in the app.

- As a user, I want to be able to set a notification for a reminder.

- As a user, I want to be able to cancel a notification for a reminder.


We found that all the user stories were accepted as completely implemented after conducting acceptance testing.


Overall, our testing approach helped us to identify and fix any issues with the Tomato, and ensure that it was functioning as intended. We were able to meet all the expected requirements and acceptance criteria, and deliver a high-quality product to our users.
